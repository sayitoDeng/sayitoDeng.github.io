'node'不是内部或外部命令

Posted on 2018-10-28

cmd提示 ‘node’ 不是内部或外部命令，也不是可运行的程序

之前项目很忙，没有时间顾得上写博客，最近闲下来了，准备搞一些自己的小项目，为以后的出路做打算。。。。（之后的博客会写到我在第一下公司的工作情况，以及项目中踩过的一些坑）

先说今天遇到的问题，感觉很莫名其妙，因为一直在出差，家里的电脑也没怎么用过，刚刚准备写demo的时候，需要NPM一些包，却出现了一些状况！

出现问题：

当我在cmd窗口运行 npm install 命令时，提示 ‘node’ 不是内部或外部命令，也不是可运行的程序或批处理文件！

产生原因：

我很确定我安装过nodeJS,那么一定就是环境变量配置出现问题！！
可是我根本就没配过啊！！！！！之前直接-v就可以用的啊！！！肯定不是我自己的原因！！！！！！（我都不信。。）
下面就写一下配置环境变量的过程，以备不时之需。

配置环境变量：

1、首先找到自己nodeJS的安装目录，我的目录是c:\Program Files\nodejs;(根据自己的目录进行修改)
2、修改环境变量，控制面板–>系统和安全–>系统–>高级系统设置–>环境变量（就按着顺序一嘛点就对了），点开环境变量之后会有用户变量和系统变量，分别找到各自的PATH，然后加上
nodeJS的安装目录，记得一定要用 ‘;’与之前的PATH分割开！！
3、重启CMD，输入npm -v，显示版本号！！完事大吉！！！！

但是！！！！！！！为毛我还是不行啊！！！！！！！！！’node’ 不是内部或外部命令，也不是可运行的程序或批处理文件！！！！！！！啊！！
之前还嚣张的敲下了 npm -v！！！！！真TM打脸啊！！！！！！！！

好！！！认了！！！！（捂脸）

plan B：
将之前的第二部进行拆分！！！！！！
（1）新建变量 NODE_PATH c:\Program Files\nodejs;
（2）修改PATH 加上%NODE_PATH%

嘿嘿！！！嘿嘿嘿嘿嘿嘿！！！完美！！（不过我想了半天也不明白为什么第一次配置环境变量没有生效，如果这篇文章在以后的某一点被某一个人看到，恰巧又是个明白人，那么请联系我！！！！！！！留下我的QQ:1095920487!!!最好写配注！！！！！！感激不尽！！！！！！！）

结尾惯例
